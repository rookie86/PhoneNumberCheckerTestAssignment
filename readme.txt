API сам выполняет нормализацию и простую очистку номера. Тестирование показало, что допустимы все следующие варианты 79139133030 +79139133030 +79139133030a a+a791a39133030a a+a891a39133030a 7(913)913-30-30

При неизвестном номере возвращает {"error":"Нет данных"} с кодом 404 
При неправильной длине возвращается сообщение {"error":"Номер должен быть 10 или 11 символов"} с кодом 200 

Исходный файл «номер телефона» / «оператор», разделенные «,»
Выходной файл так же использует запятые в качестве раздедителя.
В случае ошибки в номере или отсутствия номера в реестре вместо оператора из росреестра будет записан ответ сервера.
В случае исключения после номера и оператора будет текст исключения.

Настройка имени входного и выходного файла, адреса сервиса, задержки между обращениями к сервису осуществляется через переменные в скрипте.
